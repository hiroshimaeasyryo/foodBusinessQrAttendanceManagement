# AccessControlFoodBusiness（出勤記録）

飲食系スタッフの出勤管理用アプリ。フォーク元の入退室管理ではなく、**出勤時刻・退勤時刻・休憩時間**を定時で記録する仕様です。

## 仕様概要

- **認証**: 誕生日4桁（MMdd）入力 → 候補スタッフ表示 → スタッフ選択 → QRスキャン
- **記録**: 属性（社員 / 契約社員 / 業務委託）に応じたデフォルト時刻を記録。イレギュラー時は任意入力。
- **スプレッドシート**: 指定IDのスプレッドシートに「打刻記録」および各スタッフ名のシートへ書き込み。

詳細は [docs/implementation.md](docs/implementation.md) を参照してください。

## 構成

| ファイル | 説明 |
|----------|------|
| `Code.js` | GAS バックエンド（スタッフ取得・打刻記録・キャッシュ） |
| `index.html` | GAS で配信する画面（テンプレート） |

## セットアップ

### 1. スプレッドシート

- 使用するスプレッドシートのIDは `Code.js` の `SPREADSHEET_ID` で指定されています（仕様書のIDが既定値）。
- **スタッフDB** シート: 列 `uuid`, `name`, `birthdate`, `property`
- **打刻記録** シート: 7列（生ログ）。`タイムスタンプ`, `スタッフID`, `氏名`, `日付`, `出勤時刻`, `退勤時刻`, `休憩時間(時間)`。列はカラム名で参照。区分・勤務時間・備考は転記用シートで賄う。値は時間単位（画面入力は分のまま、例 90分→1.5）。
- **転記用** シート: 別Spreadsheetと同じ列構成（`年月`, `スタッフID`, `氏名`, `日付`, `区分`, `出勤開始`, `出勤終了`, `休憩時間(時間)`, `勤務時間(時間)`, `備考`）。確定フラグは転記先で使うため本シートには設けない。日次転記バッチ `runDailyTransfer(対象日, options)` で打刻記録と各スタッフシートの「休み」を重複なく追記し、`options.backwardDays` を指定すると過去日付の既存行を正規データと照合して更新（休み列の変更・行挿入を反映）。対象日省略時は「昨日」。定時実行には `runDailyTransferScheduled()` をトリガーに登録する（下記「転記の定時実行」参照）。
- 各スタッフ用シート: シート名は氏名の「姓」（先頭の単語）。列 `日付`, `始業`, `就業`, `休憩時間(時間)`, `休み`, `備考` など（1–2行目はヘッダー。値は時間）。「休み」は手動入力し、転記バッチで転記用シートに区分として反映される。「備考」がある場合は転記用の備考列にそのまま転記される。**スタッフシートで「始業」「就業」「休憩時間」を手動修正した場合**は、打刻記録へ反映するために **「スタッフシート → 打刻記録」の同期** を行う（下記「スタッフシートの手動修正を打刻記録に反映」参照）。

### 2. GAS デプロイ

1. [Google Apps Script](https://script.google.com/) で新規プロジェクトを作成。
2. このリポジトリの `Code.js` と `index.html` の内容をコピーするか、clasp でプッシュ。
3. `.clasp.json` の `scriptId` に上記プロジェクトのスクリプトIDを設定し、`clasp push` で反映。
4. 「デプロイ」→「新しいデプロイ」→ 種類「ウェブアプリ」、実行ユーザー「自分」、アクセス「全員」でデプロイ。
5. 表示された「ウェブアプリのURL」を控える。

### 3. 転記の定時実行（毎日「転記用」へ転記・過去の変更反映）

毎日決まった時刻に転記バッチを実行するには、GAS の時間駆動トリガーを設定する。

1. Apps Script エディタで **「トリガー」**（時計アイコン）を開く。
2. **「トリガーを追加」** で以下を設定する。
   - **実行する関数**: `runDailyTransferScheduled`
   - **イベントのソース**: 時間駆動
   - **時間ベースのトリガー**: 「日タイマー」の **午前0時〜1時**（または任意の1時間）
3. 保存すると、毎日その時刻に `runDailyTransferScheduled` が実行される。

`runDailyTransferScheduled` は「昨日」を対象日に転記し、**過去60日分**について転記用シートの既存行を打刻記録・スタッフシートの現在の内容と照合し、変更があれば上書きする（休み列の修正や後から追加した行も反映される）。**スタッフシートで手動修正した「始業」「就業」「休憩時間」は、この定期バッチの冒頭で自動的に打刻記録へ同期されてから転記される**ため、別途 `syncStaffSheetToTimestamp()` を手動実行する必要はない。対象日のみ追記したい場合は手動で `runDailyTransfer('yyyy-MM-dd')` を実行すればよい。過去反映の日数は `runDailyTransfer(対象日, { backwardDays: 90 })` のように第2引数で変更できる。

### スタッフシートの手動修正を打刻記録に反映

各スタッフ別シート（姓のシート）で「始業」「就業」「休憩時間(時間)」を手動で修正した場合、その内容を**打刻記録**シートの対応行（同じ日付・同じスタッフ）に反映するには、次のいずれかを行います。

1. **GAS から手動実行**: エディタで `syncStaffSheetToTimestamp()` を実行する。全スタッフシート・全期間が対象。
2. **日付範囲を指定**: `syncStaffSheetToTimestamp({ startDateStr: '2026-03-01', endDateStr: '2026-03-31' })` のようにオプションで対象期間を指定できる。特定の姓のみなら `syncStaffSheetToTimestamp({ staffFamilyName: '山田' })`。
3. **API から呼び出す**: ウェブアプリの URL に `?action=syncStaffSheetToTimestamp` を付与。任意で `staffFamilyName`, `startDateStr`, `endDateStr` をクエリで渡せる。

同期後、転記用シートへ反映したい場合は、続けて `runDailyTransfer(対象日, { backwardDays: N })` を実行する（転記用は打刻記録を参照するため、打刻記録を更新してから転記すると転記用にも反映される）。**定期バッチ（`runDailyTransferScheduled`）では、転記の前に同じ日付範囲で「スタッフシート → 打刻記録」の同期が自動で行われる**ため、日次トリガーを設定していれば手動同期は不要。

### 4. フロントの利用方法

- **GAS から配信**: 上記ウェブアプリのURLを**ブラウザのアドレスバーで直接開いて**ください。GAS_URL は自動で挿入されます。  
  **注意**: エディタの「プレビュー」や iframe 内ではカメラが許可されず QR が使えません。必ず「デプロイ」→「ウェブアプリのURL」を新しいタブで開いて利用してください。
- **GitHub Pages**: `standalone.html` をデプロイする際、GAS の URL を次のいずれかで設定します。
  1. **推奨**: `standalone.html` 内の `GAS_URL` の既定値（`return 'YOUR_GAS_WEB_APP_URL'` の部分）を、手順2で控えたウェブアプリのURLに書き換えてから push。
  2. ページを開くときの URL に `?gasUrl=https://script.google.com/macros/s/.../exec` を付ける（ブックマークやリンクで利用可）。
  3. `config.example.js` を `config.js` にコピーし URL を記入。`standalone.html` の jsQR の直前に行を追加: `<script src="config.js"></script>`。`config.js` を `.gitignore` に追加すると URL をリポジトリに含めずに済みます。

## 動作フロー

1. ページ読み込み時に `getStaffList` でスタッフ一覧を取得し、誕生日4桁（MMdd）ごとの候補マップをフロントで保持。
2. ユーザーが4桁入力すると、該当するスタッフのみ即表示（API 不要）。
3. スタッフを選択すると、属性に応じて勤務パターン（社員: 11-24 or 18-24、契約社員: 18-22）またはイレギュラー入力欄を表示。
4. QRコードをスキャンすると `recordTimestamp` を呼び出し、打刻記録シートとスタッフ別シートに追記。業務委託の場合は記録せず「記録なし」のみ返却。
